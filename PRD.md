# LifeCanvas · 人生画布
## 产品需求文档（PRD）v0.2 · Next.js 重构版

- 文档状态：可执行草案（用于研发启动）
- 更新时间：2026-03-03
- 负责人：产品 / 设计 / 前端 / AI 工程协同

---

## 一、产品目标

### 1.1 核心价值
将用户零散素材（照片、文稿、视频截图）自动转化为可浏览、可分享的“人生故事页”，强调情感叙事而不只是相册堆叠。

### 1.2 本轮版本目标（v0.2）
1. 架构统一迁移为 **Next.js（App Router）**。
2. 产品明确拆分为三个模块：**用例展示 / 制作 / 分享**。
3. 首个可见成果：基于 `sample_data` 生成一个老人 **高延清** 的人生样例页。
4. 页面右上角提供 `开始制作` 入口，点击后进入制作流程，可填写 API 并上传素材生成个人页面。

### 1.3 目标用户
1. 有纪念需求的普通用户（家庭、长者、子女）
2. 想制作周年/人生回顾内容的创作者
3. 需要快速出“可分享页面”的轻度内容用户

---

## 二、产品范围

### 2.1 功能模块（本期）

| 模块 | 目标 | 产出 |
|---|---|---|
| 用例展示 | 先让用户看到“成品长什么样” | 高延清人生样例页 + 首页引导 |
| 制作 | 让用户可配置 API、上传素材并生成页面 | 制作工作台（表单 + 上传 + 生成状态） |
| 分享 | 让生成结果可传播 | 分享链接、海报图、PDF（MVP可先保留按钮/占位） |

### 2.2 暂不纳入（后续版本）
1. 完整账号系统（本期可匿名项目）
2. 多主题换肤系统（本期保留单主题）
3. 复杂多人协作编辑

## 三、信息架构与路由（Next.js）

```
/                         首页（用例展示入口）
├─ /story/gao-yanqing     高延清人生样例页（基于 sample_data）
├─ /create                制作入口（API 配置 + 上传）
├─ /create/[projectId]    项目生成与预览页
└─ /share/[storyId]       对外分享页
```

### 3.1 顶部导航统一要求
1. 左侧：LifeCanvas Logo
2. 右侧：`开始制作` 主按钮（全站统一入口）
3. 辅助入口：`查看示例`

---

## 四、核心用户流程

### 4.1 访客流程（用例优先）
首页 -> 查看高延清样例页 -> 被效果吸引 -> 点击右上角 `开始制作` -> 进入制作页

### 4.2 制作流程（MVP）
填写 API 信息 -> 上传照片/文稿 -> 点击生成 -> 查看生成进度 -> 进入预览页 -> 进入分享页

### 4.3 无 API 场景
当未配置 API 或调用失败时，进入“示例模式”：
1. 使用规则模板和本地 mock 文案生成基本页面结构
2. 明确提示用户“当前为示例生成结果”

## 五、模块需求明细

### 5.1 用例展示模块

#### 5.1.1 高延清人生样例页（首个必须交付）
数据来源：
1. `sample_data/自传与作品集《清 风 霁 月》.doc`
2. `sample_data/人生轨迹照片/*`
3. `sample_data/个人肖像照片/*`

页面结构（建议）：
1. 封面：姓名、年代标签、一句人物导语
2. 人生时间线：关键节点（如出生、参军、转业、退休、文学创作）
3. 代表照片区：轨迹照 + 肖像照混排
4. 作品与精神关键词：例如“珍爱时光”“诗词创作”等
5. 总结段落：AI/模板叙事总结
6. 固定 CTA：`开始制作我的人生页面`

交互要求：
1. 右上角始终可见 `开始制作`
2. 样例页底部再次提供 `开始制作`
3. 移动端可完整浏览，不出现横向溢出

#### 5.1.2 首页
1. 展示项目定位与一句价值主张
2. 放置“高延清样例卡片”
3. 点击卡片跳转 `/story/gao-yanqing`

---

### 5.2 制作模块

#### 5.2.1 制作入口页 `/create`
分两块：
1. API 配置区
2. 素材上传区

API 配置字段：
1. `Provider`（可选项：302.ai / OpenAI Compatible）
2. `API Key`（必填，前端输入，后端安全调用）
3. `Base URL`（选填，默认 provider 标准地址）
4. `Model`（必填，例如 `gemini-1.5-pro`）

素材上传字段：
1. 人物照片（多文件）
2. 轨迹照片（多文件）
3. 文稿文件（doc/txt/pdf）
4. 情感标签（多选）
5. 生成年份（可选）

按钮与状态：
1. `开始制作`：提交并创建 `projectId`
2. 进度状态：上传中 / 解析中 / 生成中 / 完成 / 失败
3. 错误提示：API 无效、文件格式错误、超时重试

#### 5.2.2 项目生成页 `/create/[projectId]`
1. 实时或轮询显示生成进度
2. 生成结果可预览（基本区块 + 文案）
3. 提供“去分享”入口

---

### 5.3 分享模块

#### 5.3.1 分享页 `/share/[storyId]`
1. 可公开访问（无需登录）
2. 支持复制链接
3. 提供海报导出按钮（MVP可先占位）
4. 提供 PDF 导出按钮（MVP可先占位）

#### 5.3.2 分享元信息
1. 配置 Open Graph（标题、摘要、封面图）
2. 移动端分享卡片展示正常

## 六、技术架构（强制采用 Next.js）

### 6.1 前端
1. 框架：**Next.js（App Router）+ TypeScript**
2. 样式：Tailwind CSS（可配合 CSS Modules）
3. 动效：Framer Motion（首屏和区块 reveal）
4. 渲染策略：
   - 用例页：SSG/ISR
   - 制作页：CSR + Server Action / Route Handler
   - 分享页：SSR 或 ISR

### 6.2 后端能力（同仓内）
1. `app/api/*` 路由处理上传、调用模型、写入结果
2. 文件存储：
   - MVP：本地/对象存储二选一（推荐对象存储）
3. 数据存储：
   - MVP：SQLite / Supabase 任一
4. 任务机制：
   - MVP 可先同步+短任务轮询
   - 后续接队列（BullMQ/云任务）

### 6.3 AI 接入策略
1. 支持可配置 Provider（先支持 302.ai）
2. API Key 不写死在代码中
3. 前端仅提交，后端代调用模型接口
4. 统一返回结构化 JSON（事件、摘要、图注、关键词）

### 6.4 安全与隐私
1. API Key 仅会话态/安全存储，不写入公开日志
2. 上传文件大小与类型校验
3. 对外分享默认匿名化（不暴露原始文件路径）

## 八、首版交付计划

### 阶段 M1（先做样例）
1. 完成 `/story/gao-yanqing`，使用 `sample_data` 生成可浏览人生页
2. 完成全站右上角 `开始制作` 跳转 `/create`

1. 完成 `/create` API 配置 + 上传表单

### 阶段 M3（可分享）
1. 完成 `/share/[storyId]`
2. 复制链接可用
3. 海报/PDF 按钮先占位，后续补全真实导出

## 九、验收标准（本轮）

1. 打开首页可进入“高延清人生样例页”。
2. 任意页面右上角存在 `开始制作`，点击进入 `/create`。
3. `/create` 可填写 API 信息并上传文件，触发“开始制作”。
4. 生成完成后可进入分享页并复制链接。
5. 全流程在桌面端和移动端均可走通，无阻断。

## 十、版本结论

本版 PRD 将产品从“静态展示原型”升级为“可执行的 Next.js 产品方案”。  
研发优先级明确为：**先样例展示（高延清）-> 再用户制作 -> 最后分享增强**。
