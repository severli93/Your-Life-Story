## 一、产品重定义

### 1.1 名称与核心隐喻

**产品名：生命长河**

"长河"既是产品名，也是 AI 伴侣的名字。隐喻：生命是一条流动的河——有湍急、有平静、有支流汇入、有弯道转折。记录是造舟，理解是航行，呈现是让别人看见这条河的样子。

### 1.2 从"故事生成工具"到"生命记录平台"

v0.2 版本的核心是：上传素材 → AI 生成故事页。

v2.0 重新理解产品的核心价值：

| 维度 | v0.2 | v2.0 |
|---|---|---|
| 核心动作 | 生成 | 记录 + 理解 + 呈现 |
| 用户关系 | 一次性制作 | 持续陪伴 |
| 核心交付物 | 故事页面 | 情绪成长档案 + 人生可视化 |
| AI 角色 | 生成工具 | 对话伴侣（长河） |

"生成人生长河"（可视化故事页）是产品的**输出功能之一**，而非产品全部。

## 二、产品结构

```
生命长河
│
├── 长河（AI 对话伴侣）          ← 日常入口，持续使用
│   ├── 情绪觉察对话
│   ├── 记忆提取引导
│   └── 每周/每月洞察
│
├── 记忆层                       ← 数据沉淀
│   ├── 模式 A：整理过去的记忆（导入历史）
│   └── 模式 B：从现在开始记录（日/周/月/年）
│
└── 生成人生长河                  ← 阶段性产出
    ├── 年度报告（最轻量）
    ├── 专题故事页（爱情/成长/家族等）
    └── 完整人生时间轴（长期积累后）
```

## 三、用户旅程

### 3.1 新用户首次进入

```
首页 → 了解"生命长河"是什么
  → 两种起点：
    A. 我想整理过去的记忆（长河引导对话，逐步建立时间轴）
    B. 我想从现在开始记录（选择记录频率，长河定期陪伴）
  → 也可以先看样例（高延清的故事）
  → 被效果吸引 → 开始自己的故事
```

### 3.2 日常使用（持续用户）

```
打开长河 → 今日情绪对话（5-15分钟）
  → 自动记录情绪节点
  → 每周日：长河推送本周洞察
  → 积累一定记忆后：提示可生成当前阶段的"人生长河片段"
```

### 3.3 创作与分享

```
查看我的记忆库 → 选择时间范围/主题
  → 生成可视化故事页
  → 预览 → 分享/下载
```

## 五、功能模块详细需求

### 5.1 首页 `/`

**目标：** 5秒内理解产品价值，找到自己的入口。

**内容区块：**
1. Hero：产品名 + 一句话价值主张 + 两个主入口（开始记录 / 看样例）
2. 产品核心循环说明（3步：对话 → 沉淀 → 可视化）
3. 高延清样例卡片（视觉驱动转化）
4. 故事类型展示（年度回顾/爱情/成长等）

**不需要：** 复杂的功能列表、价格、注册墙

---

### 5.2 长河对话伴侣 `/companion`

**核心能力：**

| 场景 | 长河的行为 |
|---|---|
| 日常情绪 | A·I·R 三步引导（感受→影响→请求） |
| 情绪不明 | 身体信号定位 + 情绪命名 |
| 回忆提取 | 时间锚定 → 场景还原 → 情绪挖掘 → 当下连接 |
| 高情绪时刻 | 暂停技术 + 呼吸引导 |
| 比较焦虑 | 转化框架（标准 vs 基准） |
| 拖延 | 同理心脚本（没有指责） |

**记忆模式选择（首次进入）：**
- 🗂 整理过去：AI 引导对话，逐步提取历史记忆
- 📔 从现在记录：选择粒度（日/周/月/年），长河定期陪伴

**界面布局：**
- 左侧：时间轴树 + 情绪热力图（侧边栏）
- 主区：对话界面 + 情绪温度条
- 底部：输入框（"此刻的心情..."）

**技术：** 302.ai proxy × claude-sonnet-4-6，流式输出

---

### 5.3 记忆库 `/memories`

**时间轴视图：**
- 按年/月折叠展示记忆节点
- 每个节点：时间 + 情绪标签 + 关键摘要 + 点击展开对话
- 年度总结卡（主题词 + 情绪主色调）

**洞察视图：**
- 本周情绪分布（CSS 柱状图）
- 情绪走势（按星期几）
- AI 观察卡片

---

### 5.4 生成人生长河 `/story/new` → `/story/[id]`

这是产品的"结晶时刻"——将积累的记忆，转化为可视化叙事。

**故事类型（当前支持）：**
| 类型 | 素材来源 | 说明 |
|---|---|---|
| 年度回顾 | 长河对话记录 | 最轻量，无需额外上传 |
| 人生时间轴 | 对话记录 + 手动上传照片/文稿 | 高延清样例展示的类型 |
| 爱情故事 | 照片 + 文字描述 | 专题类 |
| 成长记录 | 照片时间线 | 专题类 |

**创建流程：**
```
选择故事类型 → 选择时间范围（从记忆库中选）
→ 可追加上传照片/文稿
→ 配置 API（Provider / Key / Model）
→ 生成 → 进度展示 → 预览 → 发布/分享
```

**生成页面结构（参考高延清样例）：**
- 封面：姓名、年代、人物导语
- 时间线：关键节点滚动叙事（GSAP ScrollTrigger）
- 照片区：轨迹照 + 肖像 混排
- 关键词云：情绪/精神标签
- AI 叙事总结段落
- CTA：开始制作我的故事

---

### 5.5 分享页 `/share/[id]`

- 无需登录可访问
- Open Graph 配置（标题/摘要/封面图）
- 复制专属链接
- 下载 PDF（MVP 先占位）
- 发布朋友圈图片（MVP 先占位）

## 六、技术架构

### 6.1 前端
- 框架：Next.js 14（App Router）+ TypeScript
- 样式：Tailwind CSS + CSS Modules
- 动效：Framer Motion（页面过渡）+ GSAP ScrollTrigger（故事页滚动叙事）
- 字体：Nunito（英文圆润主字体）+ 马善政/Noto Serif SC（中文）

### 6.2 AI 接入
- Provider：302.ai（可配置）
- Model：claude-sonnet-4-6（默认）
- SDK：openai npm package（兼容 302.ai 接口）
- 对话：流式输出（Stream）
- 故事生成：结构化 JSON 输出

```typescript
// 统一 LLM 客户端
const client = new OpenAI({
  apiKey: process.env.API_302_KEY,
  baseURL: 'https://api.302.ai/v1',
});
```

### 6.3 数据存储（MVP）
- 对话历史：本地 JSON 文件（`/data/sessions/`）
- 记忆节点：`/data/memories.json`
- 情绪记录：`/data/emotions.json`
- 用户上传文件：本地 `/public/uploads/` 或对象存储

### 6.4 数据模型

```typescript
// 情绪记录节点
interface EmotionRecord {
  sessionId: string;
  timestamp: string;
  emotionTags: string[];
  emotionTemperature: number;      // 0-10
  bodySignals?: string[];
  triggers?: string[];
  keyQuote: string;
  skillUsed: string;               // 使用了哪个对话技能
  granularity: 'day' | 'week' | 'month' | 'year';
}

// 记忆节点（已处理的长期记忆）
interface MemoryNode {
  memoryId: string;
  timePeriod: string;              // "2025-Q4", "2024年"
  emotionTags: string[];
  keyMoments: string[];
  coreQuote: string;
  currentConnection?: string;      // 与当下的连接
  mediaUrls?: string[];            // 关联照片/文稿
  createdAt: string;
}

// 故事生成结果
interface StoryResult {
  storyId: string;
  personName: string;
  storyType: 'annual' | 'timeline' | 'love' | 'growth';
  timeline: TimelineEvent[];
  summary: string;
  keywords: string[];
  shareId: string;
  createdAt: string;
}
```

## 九、架构决策记录（2026-03-04）

### 9.1 首页入口
温情风格，展现具体人生片段，强调"记录"与"生成报告"。
两个核心入口并列：
- **开始对话** → 进入长河对话伴侣
- **查看样例 / 直接生成年度人生** → 进入样例页或快速体验

### 9.2 故事生成流程
- **允许直接上传素材**（不强制先对话），上传过程中由长河以对话形式引导补充
- **支持累计导入**：Notion、其他日记 App 等外部内容
- **支持生成历史日历**，并支持生成后继续对话
- 上传 + 对话功能集成在同一 Agent 模块中

### 9.3 账号与登录
- 做账号 mock 效果，实际以**游客身份**也可完整试用
- 点击"生成报告"时触发引导："转发获得 Token"（病毒传播机制）
- MVP 阶段做成 Demo，本地可完整体验全部功能

### 9.4 样例策略
三类样例覆盖不同用户：
| 样例 | 主角 | 风格 | 状态 |
|---|---|---|---|
| 高延清的故事 | 老人（85岁）| 深色·史诗感·滚动叙事 | ✅ 已完成 |
| 小棠的第一千天 | 婴儿（0-3岁）| 暖色·视频感·自动播放 | 🔧 当前任务 |
| 年轻人年度报告 | 年轻人（25岁）| 数据感·Spotify Wrapped | 📋 待规划 |

### 9.5 多端策略
两端都需要完整体验，同步开发：
- **移动端**：单列全屏，手势滑动，对话优先
- **桌面端**：宽布局，更多信息密度，故事展示优先
