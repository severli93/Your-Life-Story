# 高延清样例页 视觉与动效规格 v1.0

- 适用范围：Next.js 版本的 `首页 / 故事页 / 制作页`
- 视觉方向：Spotify Wrapped 节奏 + Lupi 注释层 + Felton 极简信息排版 + Scrollytelling 纵向叙事
- 人设关键词：时代记忆、克制、坚韧、文学性

---

## 0. 全局规范

### 0.1 视觉 Token
1. 主背景：`#08080E`
2. 次背景：`#13131F`
3. 高亮金：`#D4A853`
4. 点缀米粉：`#E8C4B8`
5. 文本主色：`#EDE8E0`
6. 弱文本：`#A09888`
7. 圆角：卡片 `22-24px`，胶囊按钮 `100px`

### 0.2 字体层级
1. 标题：Playfair Display
2. 叙事正文：EB Garamond（可斜体）
3. UI：DM Sans

### 0.3 动效曲线与时长
1. 主过渡：`cubic-bezier(.25,.46,.45,.94)`，`400-700ms`
2. 叙事 reveal：`650-850ms`
3. 关键节点高亮：`240-320ms`

### 0.4 滚动叙事主线
1. 隐形纵向时间轴贯穿页面
2. 每屏只突出一个核心信息点
3. Section 进入时触发“时间轴节点高亮 + 内容渐入”

## 1. 首页规格（Home）

### 1.1 信息目标
1. 5 秒内建立“AI 生成人生页面”的认知
2. 强引导进入“高延清样例页”
3. 顶部固定 CTA：`开始制作`

### 1.2 区块结构
1. Hero：价值主张 + 双 CTA（查看样例 / 开始制作）
2. 样例卡片：高延清人物卡（肖像 + 生平标签 + 入口）
3. 能力摘要：上传素材 -> AI 解析 -> 生成分享

### 1.3 动效规格
1. 首屏星空背景：低速粒子漂移（常驻，低对比）
2. Hero 文案：分层 stagger（延迟 `120ms`）
3. 样例卡片：滚动进入时 upward reveal + 金色边框呼吸光

### 1.4 技术建议
1. `Framer Motion`：首屏文案/卡片 reveal
2. `GSAP ScrollTrigger`：样例区滚动触发
3. `Next/Image`：首屏图片优化

## 2. 故事页规格（/story/gao-yanqing）

### 2.1 信息目标
1. 用时间线讲清高延清的人生阶段
2. 用照片与文稿并列，形成“图像 + 文字证据”
3. 每段末尾给出进入制作流程的 CTA

### 2.2 区块结构
1. 封面段：姓名、年代、人物一句话
2. 生平时间线段：出生 -> 抗美援朝 -> 军旅/转业 -> 退休创作
3. 回忆图像段：人生轨迹照片 + 手绘注脚连线
4. 文稿洞察段：关键词、字数、章节密度
5. 人物总结段：AI 叙事段落 + `开始制作我的页面`

### 2.3 动效规格
1. 时间线节点：进入视口触发点亮，上一节点降亮
2. 回忆图像：几何遮罩切换主图，注脚延迟 `180ms` 出现
3. 文稿指标：数字滚动动画（0 -> 目标值）
4. 段落切换：每段仅一个主动画，防止信息拥挤

### 2.4 图像与注释规范（Lupi 风）
1. 注脚长度：12-28 字
2. 每张图最多 2 条连线，避免视觉噪音
3. 注释层用 SVG Overlay，不写死到位图

### 2.5 技术建议
1. `GSAP + ScrollTrigger`：时间线与章节切换
2. `SVG`：注脚连线与节点
3. `D3.js`：文稿统计图（词频/章节密度）

## 3. 制作页规格（/create）

### 3.1 信息目标
1. 让用户明确“填写 API + 上传素材 + 开始制作”
2. 尽量降低操作焦虑（分步、可见状态）
3. 失败可恢复（错误提示 + 重试）

### 3.2 布局结构
1. 左侧（或上方）：API 配置
2. 右侧（或下方）：素材上传（肖像/轨迹/文稿）
3. 底部固定操作区：`开始制作`

### 3.3 表单与状态
1. API 字段：Provider / API Key / Base URL / Model
2. 上传字段：人物照、轨迹照、文稿、情感标签
3. 状态流：`idle -> uploading -> parsing -> generating -> done|failed`

### 3.4 动效规格
1. 上传成功：缩略图淡入 + 勾选标记弹出
2. 开始制作后：全局进度环 + 阶段文案切换
3. 失败提示：按钮轻震 + Toast 错误信息

### 3.5 技术建议
1. `React Hook Form` + `zod`：表单验证
2. `Server Actions` 或 `Route Handlers`：提交与调用后端
3. 进度轮询：`SWR` 或 `react-query`

## 4. 音乐功能规格（全站）

### 4.1 目标
1. 提供“纪念感”背景音乐，默认关闭
2. 用户可在页面顶部一键开/关
3. 记住用户偏好（下次进入保持状态）

### 4.2 交互规则
1. 控件位置：导航右上角（全站可见）
2. 文案状态：`音乐：关` / `音乐：开`
3. 点击开启后播放 loop；再次点击立即暂停
4. 浏览器禁止自动播放时，给出明确提示并保持关闭

### 4.3 音频规范
1. 建议时长：30-60 秒循环
2. 默认音量：`0.30-0.35`
3. 音源格式：优先 `mp3`，兼容可保留 `wav`
4. 当前样例音源：`assets/audio/gao-theme.wav`

### 4.4 无障碍要求
1. 按钮使用 `aria-pressed`
2. 键盘可触发（Enter/Space）
3. 文案必须可见，不能只用图标表达状态
